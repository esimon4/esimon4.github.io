Feature #1 ‚Äî Favorites
After the HTML and CSS were close to completion, I began work on the favorites feature. I started by storing all of the blog info (title, author, relevant keywords) as objects in a universal array. Then, I created an empty array for favorited blogs. The HTML was set up so each blog post had a corresponding heart icon with a unique ID.

When a blog‚Äôs heart icon is pressed, the code first checks to see if that blog is already stored into favorites. If it is not, the blog‚Äôs corresponding array object is pushed to the favorites array. That favorites array is then saved into local storage. Finally, the empty heart icon is replaced with a filled heart to improve the UX.

Users can remove a blog from their favorite post section by clicking the filled-in heart icon. When a user presses the filled-in heart icon, the code parses through the favorites array, finds where that specific blog is, and removes it.

Every time the page is loaded, the code parses local storage to find existing favorites, re-stores those blogs into the favorites array, and fills in all of the hearts of the favorite blogs that were in storage. The end result came out like this:
Feature #2 ‚Äî Search
The next feature I built out was the search function. Each blog subpage had its own search bar with a unique HTML ID. I began by creating a simple event listener that, when the user clicks the ‚Äòrun search‚Äô icon, passes the word(s) they typed through an external function.

The external function then takes the search phrase and, using an indexOf() method, checks it against the title, authors, and keywords stored into each blog‚Äôs respective array object.

If the search phrase matches any blog‚Äôs title, author, or keywords, the code printed that blog out in the search results. If there is no match, and the indexOf() returned a -1 value, an error message prints. The favorites feature integrates seamlessly with the search function ‚Äî users can favorite and un-favorite blog posts directly from their search results.

Then I created 18 variables (one for each blog on the website) and set their values to 0. The idea was that every time a user‚Äôs answer correlated to a certain blog subject (for example, they answered ‚Äòyes‚Äô to a question asking if they were in need of housing and there are 2 blog posts offering advice for off-campus housing searches) the code would add a value of ‚Äú1‚Äù to the blog‚Äôs respective variable.
